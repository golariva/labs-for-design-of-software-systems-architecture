# Лабораторная работа №1
## Тема:
Формулирование требований к программной системе
## Цель работы:
Научиться анализировать поставленную задачу, формулировать функциональные и нефункциональные требования к проектируемой системе.
## Перечень заинтересованных лиц
1. Сотрудники учебного офиса (основные пользователи системы, которые загружают, хранят и распределяют документы студентов).
2. Руководитель учебного офиса (контролирует соблюдение номенклатуры дел, отвечает за качество хранения и архивирование).
3. Новые сотрудники учебного офиса (нуждаются в автоматизации, упрощающей адаптацию и исключающей ошибки при классификации документов).
4. Преподаватели (работают с документами по ВКР, курсовым, отчетам и ожидают, что эти документы будут быстро и качественно выгружаться учебным офисом).
5. Архив университета (принимает готовые дела, требует корректной категоризации и соблюдения сроков хранения).
## Перечень функциональных требований
1. Автоматическая категоризация документов (система определяет категорию по номенклатуре - тип документа, содержимое, ключевые слова).
2. Генерация стандартизированного имени файла (система присваивает имя по шаблону: <Категория> <Тип документа> <Дата> <Доп. поля>).
3. Автоматическое размещение файла (файл сохраняется в правильную директорию, сформированную на основе номенклатуры).
4. Ручной выбор категории (если автоматическая классификация неверна, пользователь может выбрать категорию вручную).
5. Поиск по метаданным (поиск по: названию, типу документа, категории, дате, ключевым словам).
6. Просмотр структуры номенклатуры (отображение перечня дел, их индексов, сроков хранения и описаний).
7. Массовая загрузка файлов (поддержка загрузки пакета файлов).

## Перечень сделанных предположений
1. Номенклатура дел предоставлена учебным офисом и фиксирована на учебный год.
2. Все документы поступают в электронном виде (PDF, DOCX, JPG).
3. Структура папок на диске H не является формально утвержденной системой.
4. Сотрудники размещают файлы вручную и используют нестандартизированные имена.
5. Система SmartReg нестабильна при обновлениях, что требует хранения документов.
## Перечень нефункциональных требований
1. Производительность (время классификации одного файла не должно превышать 2 секунд).
2. Надежность (система обязана корректно работать при временном отсутствии доступа к сетевому диску).
3. Удобство использования (интерфейс должен быть понятен сотрудникам без технической подготовки).
4. Безопасность (доступ к системе должен иметь только персонал учебного офиса, так как документы содержат персональные данные).
5. Масштабируемость (возможность добавления новых категорий и изменения номенклатуры без переписывания системы).
