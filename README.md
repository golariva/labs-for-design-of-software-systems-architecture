# Разработка приложения для категоризации документов учебного офиса университета
Учебный офис университета работает с большим объемом документов студентов: ВКР, курсовые, заявления, справки, приказы, данные об успеваемости и другие материалы. Документы хранятся на сетевом диске в виде вложенных папок, и их распределение выполняется вручную.
Из-за отсутствия автоматизации возникают проблемы: поиск нужного документа затруднен, единый стандарт именования не соблюдается, а новые сотрудники долго адаптируются к структуре хранения.
Дополнительно встречаются сложности с выгрузкой файлов из Smart LMS и потерей данных после обновлений системы SmartReg.
## Цель
Создать систему, которая автоматизирует классификацию, именование и размещение электронных документов согласно номенклатуре учебного офиса.
##Пользователи:
* сотрудники учебного офиса - ежедневно работают с документами, ищут, сохраняют и распределяют файлы;
* руководитель учебного офиса - отвечает за соблюдение номенклатуры, корректность хранения и отчетность;
* новые сотрудники - нуждаются в понятной и автоматизированной системе для быстрого погружения;
* преподаватели - передают ВКР, курсовые, проекты, ожидают, что офис сможет оперативно предоставлять документы;
* архив университета - принимает итоговые дела, важна корректная категоризация и сроки хранения.
## Требования:
* сотрудники загружают документы в систему;
* система автоматически определяет категорию документа согласно номенклатуре;
* система предлагает стандартизированное имя файла (по шаблону категории);
* документ автоматически размещается в соответствующей папке в сетевом диске;
* пользователь может вручную выбрать категорию, если предложенная системой некорректна;
* поддерживается поиск по категории, типу документа, дате, ключевым словам;
* выгруженные из LMS файлы можно массово загрузить для автоматической сортировки;
* система визуализирует структуру номенклатуры (индексы, сроки хранения, описания);
* руководитель может редактировать параметры отдельных категорий.
## Дополнительный контекст:
* документы учебного офиса содержат личные данные студентов, что требует строгого соблюдения порядка хранения;
* SmartReg регулярно обновляется, и сотрудники сталкиваются с потерей данных и необходимостью повторного ввода информации;
* ВКР, курсовые и проекты хранятся в Smart LMS, но их приходится вручную скачивать и размещать на диске H - система должна упростить и ускорить процесс;
* университет не использует автоматизированные инструменты категоризации электронных документов, текущая схема опирается только на опыт сотрудников;
* автоматизация необходима для повышения скорости работы, снижения количества ошибок и обеспечения единообразия хранения.
